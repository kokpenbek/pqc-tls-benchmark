# NGINX server configurations

Two NGINX configurations are used for the benchmark:

- `classic.conf` — baseline with `Groups X25519`.
- `hybrid.conf` — hybrid post-quantum with `Groups X25519MLKEM768`.

Both files are otherwise identical, so the key exchange algorithm is the only variable across measurements.

## Configuration notes

- The key exchange group is pinned via `ssl_conf_command Groups ...`. There is no fallback to other groups, so each configuration isolates a single algorithm.
- `ssl_session_cache off` and `ssl_session_tickets off` disable session resumption, forcing every connection to perform a full TLS handshake. This is essential for meaningful handshake-time and TTFB measurements.
- Only TLS 1.3 is enabled (`ssl_protocols TLSv1.3`).

## Endpoints

Four HTTP endpoints are exposed by both configurations for different measurement scenarios:

| Endpoint | Response | Purpose |
|----------|----------|---------|
| `/api/direct` | 2-byte JSON | Handshake-dominated measurement; minimal application payload. |
| `/api/payload/1kb` | 1 KB random binary | Small-payload TTFB. |
| `/api/payload/100kb` | 100 KB random binary | Medium-payload throughput. |
| `/api/payload/1mb` | 1 MB random binary | Large-payload throughput. |

Payload files are generated by `setup/generate_payloads.sh` and served from `/var/www/example.com/`.
